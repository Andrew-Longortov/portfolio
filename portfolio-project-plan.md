## Milestone 1 — HTML-скелет Главной

**Задачи:**
1. `index.html`: семантический каркас — `header` с навигацией, `main`, `footer`.
2. Секция Hero: заголовок, короткий подзаголовок, кнопка-ссылка «Смотреть проекты».
3. Секция About: фото/аватар + список фактов о себе (имя, локация, email, статус).
4. Секция Tech Stack: пока просто список технологий текстом (иконки — на этапе CSS).
5. Секция Projects: 4 карточки-заглушки (название + короткое описание), ссылки уже сейчас веди на `/projects/*/index.html`.
6. Секция Contact: email и соцсети.

**Готово, когда:** страница открывается в браузере, вся информация читаема без стилей, а в DevTools → Elements иерархия тегов понятна с первого взгляда.

**Вопросы для самопроверки:** кнопки — это `<button>` или `<a>`, а не `<div onclick>`? У всех изображений есть `alt`? Заголовки идут по порядку (`h1` → `h2` → `h3`, без пропусков)?

## Milestone 2 — CSS: базовая стилизация и адаптив

**Задачи:**
1. CSS-reset или normalize (можно минимальный свой: `box-sizing: border-box`, обнуление отступов).
2. Задать типографику и цветовую палитру через CSS-переменные (`:root { --bg: ...; --accent: ...; }`).
3. Разложить Hero и карточки проектов через Flexbox/Grid.
4. Добавить media queries хотя бы под 3 брейкпоинта: мобильный / планшет / десктоп.
5. hover/focus-состояния для кнопок и карточек (transition — плавно, не резко).

**Готово, когда:** сайт нормально выглядит и при 375px ширины, и при 1440px, без горизонтального скролла и наезжающих друг на друга элементов.

## Milestone 3 — JS: Калькулятор

**Что понадобится:** обработчики кликов, работа с числами/строками, чистые функции для арифметики.

**На что обратить внимание:**
- Как хранить текущий ввод и предыдущее значение (например, в переменных состояния, а не читать прямо из DOM каждый раз).
- Деление на ноль — что должно происходить?
- Цепочка операций (`5 + 3 - 2 =`) — как обрабатывать несколько операторов подряд?
- Кнопка очистки (`C`/`AC`) и, по желанию, удаление последнего символа.

**Готово, когда:** калькулятор корректно считает +, −, ×, ÷, не ломается на дробных числах и не падает в консоли при вводе `0÷0`.

## Milestone 4 — JS: Генератор паролей

**Что понадобится:** работа со строками, `Math.random()`, чекбоксы и range-инпут, Clipboard API (`navigator.clipboard.writeText`).

**На что обратить внимание:**
- Наборы символов (буквы верхний/нижний регистр, цифры, спецсимволы) как отдельные строки, которые собираются в один пул в зависимости от выбранных чекбоксов.
- Что делать, если пользователь снял все чекбоксы — как избежать пустого пула?
- Кнопка «Скопировать» должна давать понятную обратную связь (например, временная смена текста кнопки).

**Готово, когда:** пароль генерируется нужной длины, учитывает все включённые опции, копируется в буфер обмена по клику.

## Milestone 5 — JS: Конвертер валют

**API:** можно взять open.er-api.com — бесплатный, без ключа (или exchangerate-api.com, если хочешь потренироваться работать с API-ключом).

**Что понадобится:** `fetch`, `async`/`await`, `try/catch` для ошибок сети.

**На что обратить внимание:**
- Что показывать пользователю, пока данные загружаются (лоадер/текст «Загрузка…»)?
- Что показывать при ошибке (сеть недоступна, невалидный ответ)?
- Как избежать лишних запросов — например, не дёргать API на каждое нажатие клавиши, а по кнопке или с debounce.

**Готово, когда:** конвертер показывает актуальный курс, корректно обрабатывает как минимум одну ошибку (например, отключи себе интернет и проверь, что сайт не падает, а показывает сообщение).

## Milestone 6 — JS: Прогноз погоды

**API:** OpenWeatherMap (нужна бесплатная регистрация и API-ключ).

**Важный нюанс про ключ:** на чисто статичном сайте (GitHub Pages и т.п.) ключ API оказывается виден в исходном коде — это ограничение таких проектов, для pet-проекта это нормально, но стоит знать: в реальном продакшене такие ключи прячут за собственным бэкендом-прокси. Можно упомянуть это как заметку в README проекта — плюс в карму на собеседовании, если спросят.

**Что понадобится:** `fetch`, рендер данных в DOM, по желанию — Geolocation API (`navigator.geolocation.getCurrentPosition`).

**Готово, когда:** по названию города показывается текущая погода (температура, влажность, ветер), обработана ситуация «город не найден».
