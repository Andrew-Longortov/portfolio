# Roadmap: путь в Fullstack-разработчика

Этот план построен вокруг твоей же идеи — сначала железно закрыть HTML/CSS/JS на практике (портфолио-сайт с 4 pet-проектами), а затем осознанно выбрать бэкенд-стек. Сроки условные, из расчёта 1–2 часа занятий в будни + больше по выходным. Если получается быстрее или медленнее — это нормально, скорость у всех разная.

## Как этим пользоваться

Каждый этап = тема + практика + «чек-поинт» (как понять, что можно двигаться дальше). Практика почти везде привязана к твоему же портфолио, чтобы не распыляться на десятки несвязанных упражнений.

---

## Этап 0. Подготовка окружения (2–3 дня)

- Установить VS Code + расширения: Live Server, Prettier, ESLint, автозакрытие тегов.
- Базовый Git: `init`, `add`, `commit`, `push`, `pull`, `branch`, `.gitignore`, чтение `git status`/`git log`.
- Терминал: `cd`, `ls`, `mkdir`, `rm`, `touch` — минимум, чтобы не бояться консоли.
- Завести аккаунт на GitHub и репозиторий под портфолио.

**Чек-поинт:** можешь создать репозиторий, закоммитить файл и запушить его на GitHub без подглядывания в шпаргалку.

---

## Этап 1. HTML (1–2 недели)

**Темы:** семантические теги (`header`, `nav`, `main`, `section`, `article`, `footer`), формы и валидация (`input`, `label`, `required`, `pattern`), таблицы, медиа (`img`, `picture`, `video`), доступность (`alt`, `aria-*`, порядок табуляции), базовые meta-теги.

**Практика:** сверстать HTML-структуру Главной страницы портфолио (Hero, About, Tech Stack, Projects, Contact) — вообще без CSS. Только смысловая разметка.

**Ресурсы:**
- MDN: «HTML basics» / «Learn HTML»
- freeCodeCamp — «Responsive Web Design» (первая часть, до CSS)
- htmlacademy — бесплатные вводные курсы

**Чек-поинт:** страница открывается в браузере, вся информация читаема без единого стиля, в DevTools → Elements иерархия тегов логична и без «div-супа».

---

## Этап 2. CSS (2–4 недели)

**Темы:** box model, селекторы и специфичность, Flexbox, Grid, адаптивная вёрстка (media queries, mobile-first), псевдоклассы/псевдоэлементы, transitions/animations, CSS-переменные. Опционально — SASS/SCSS (даёт вложенность, миксины, переменные до появления нативных CSS variables — полезно понимать, раз встречается в реальных проектах).

**Практика:** стилизовать все секции портфолио, сделать адаптив под мобильный/планшет/десктоп, добавить hover-эффекты на кнопках и карточках проектов.

**Ресурсы:**
- MDN: «Learn CSS»
- Flexbox Froggy, Grid Garden (игровые тренажёры)
- CSS-Tricks — «A Complete Guide to Flexbox / Grid»
- freeCodeCamp CSS-модули

**Чек-поинт:** портфолио выглядит презентабельно и на телефоне, и на десктопе — без единой строчки JavaScript.

---

## Этап 3. JavaScript (4–6 недель)

**Темы:**
- Базовый синтаксис: `let`/`const`, типы данных, условия, циклы, функции.
- DOM: `querySelector`, `addEventListener`, изменение контента и классов.
- ES6+: стрелочные функции, деструктуризация, spread/rest, модули (`import`/`export`), шаблонные строки.
- Асинхронность: Promises, `async`/`await`, Fetch API.
- Валидация форм на JS, `localStorage`.

**Практика — твои 4 pet-проекта, каждый закрывает свою тему:**

1. **Калькулятор** — события кликов, работа с числами и строками, чистые функции, обработка деления на ноль.
2. **Генератор паролей** — работа со строками, `Math.random`, чекбоксы/range-инпуты, Clipboard API.
3. **Конвертер валют** — `fetch` к внешнему API (например, open.er-api.com — бесплатный, без ключа), `async`/`await`, обработка ошибок сети.
4. **Прогноз погоды** — `fetch` к OpenWeatherMap, рендер данных в DOM, по желанию — Geolocation API.

**Ресурсы:**
- javascript.info — лучший бесплатный учебник по JS, от основ до асинхронности
- MDN: «Learn JavaScript»
- freeCodeCamp — «JavaScript Algorithms and Data Structures»

**Чек-поинт:** все 4 pet-проекта рабочие и встроены в `/projects`.

---

## Этап 4. Деплой и Git-flow (2–3 дня)

- Задеплоить портфолио на GitHub Pages, Netlify или Vercel.
- Написать нормальный `README.md` с описанием и скриншотом.
- (Опционально) подключить свой домен.

**Чек-поинт:** сайт доступен по публичной ссылке — можно класть в резюме и отклики.

---

## Развилка: Python (Django/FastAPI) vs Node.js

После этапа 4 у тебя уже полноценный frontend-скилл, годный для трудоустройства junior frontend. Дальше — выбор бэкенда.

**Node.js (Express / NestJS):**
- Тот же язык (JS/TS), что и на фронте — ниже порог входа сразу после этапа 3.
- Огромная экосистема npm, много вакансий, особенно в стартапах.
- Плюс: пишешь fullstack на одном языке.

**Python (Django или FastAPI):**
- Django — «батарейки в комплекте»: admin-панель, ORM, аутентификация из коробки. Хорош для быстрых MVP и монолитных проектов.
- FastAPI — лёгкий, асинхронный, отлично для API и микросервисов, современный подход через Pydantic-типизацию.
- Плюс: Python пригождается за пределами веба — автоматизация, анализ данных, если вдруг туда потянет.

**Рекомендация:** оба стека одинаково востребованы на рынке РФ/СНГ, выбор — вопрос личных предпочтений, а не «что круче». Если хочется быстрее почувствовать себя fullstack-разработчиком на одном языке — Node.js. Если интересен более «традиционный» бэкенд плюс бонус в виде Python вне веба — Django/FastAPI. Можно попробовать оба на маленьких пет-проектах (по 1 неделе) и уже тогда решить осознанно.

## Этап 5. Бэкенд и база данных (6–10 недель, после выбора стека)

**Темы:**
- Основы HTTP: методы, статус-коды, заголовки, REST.
- Базы данных: SQL-основы (SQLite/PostgreSQL), работа через ORM.
- Аутентификация/авторизация (сессии или JWT).
- Стек Node: Express или NestJS + PostgreSQL/MongoDB.
- Стек Python: Django (ORM, admin, views/templates) или FastAPI (Pydantic, async, SQLAlchemy).

**Практика:** превратить один pet-проект в fullstack. Хороший кандидат — раздел `/blog` из твоей структуры: CRUD статей (создать/прочитать/обновить/удалить) через собственный API и базу данных вместо статичного HTML.

---

## Этап 6. Что дальше

- TypeScript (пригодится в любом стеке).
- Фреймворк фронтенда — React или Vue (не обязателен для fullstack на своём бэкенде, но сильно повышает конкурентоспособность резюме).
- Тестирование (Jest для JS/TS, Pytest для Python).
- Основы Docker.
- CI/CD basics (GitHub Actions).

## Ориентировочный тайминг

| Этап | Содержание | Срок |
|---|---|---|
| 0 | Подготовка окружения | 2–3 дня |
| 1 | HTML | 1–2 недели |
| 2 | CSS | 2–4 недели |
| 3 | JavaScript + 4 pet-проекта | 4–6 недель |
| 4 | Деплой | 2–3 дня |
| 5 | Бэкенд + БД | 6–10 недель |
| 6 | Углубление (TS, фреймворк, тесты, Docker) | постоянно |

Итого до полноценного fullstack-джуна — примерно 4–6 месяцев при регулярных занятиях. Портфолио из этого плана закрывает этапы 0–4 полностью.
